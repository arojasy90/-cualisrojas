Heuristic Analysis

Heuristic Value Conclusions
Visibility Instant response when hovering over items (Carousel of articles stopping, Color change of link)
Match between System and the Real World Site used journalistic jargon "Quill." However, when cursor was placed over the word, the site instantly provided further clarification for individuals who may not be aware of the meaning of the "Quill" reference by listing ---> Editorials.
User Control and Freedom Site lacks a "home" function to return to the main page of the website. It is not intuitive to click on the website's header image to return to the main landing page
Consistency and Standards Articles appears multiple times across various site sections. There is no real explanation of the article's inclusion in the multiple sections.
Error Prevention No error prevention is visible such as "you are now leaving this site" when users click on the hyperlinks provided in the articles.
Recognition Rather then Recall Limited features of navigating the webpage. For example there is no return to the homepage feature or explore similar articles. The only feature available is the "^" return to the top button
Flexibility and Efficiency of use No advance features available to "expert users." Little to no features provided for general population users.
Aesthetic/Minimalist Design The site is extremely distracting. Multiple items are moving a different rates and/or are flashing. These elements causes a competition for the user's attention. The page lacks a "responsive web design" (RWD) that adapts the page to various screen sizes.
Help Users Recognize, Diagnose, and Recover from Errors There are no user diagnostic tools or messages available. It appears that if an error is made, the user's only course of action is to reload the page.
Help and Documentation No place to contact or receive support.

Part 1 Evaluative Summary

Strengths

  1. Responsive interface where user input stimulates immediate output (i.e. cursor over Archive link changes the text colors
  2. The site does a great job with providing clarifying details for unfamiliar terminology (Quill-->editorials)

Weakness

  1. Too many interactive displays throughout the web interface. It is very distracting to the user who is seeking information.
  2. There is no error prevention measures to prevent users from unintentionally executing actions.

Comparative Analysis of Heuristic review and mini-usability test

As stated in the directions the mini-usability test was broken into two separate parts. I asked the participant to explore and take note of the various elements on the webpage. After the 5 minute mark, I asked my participant to reflect on the web page layout in comparison to other webpages that they frequently visit. As they compared the site's capabilities, I ask them which specific websites they were utilizing for the comparative analysis. They stated Newsweek and a Pokémon Database. The participant stated that website lacked a way to easily navigate through content. Further, the participant indicated when they have difficulty finding specific results when they applied the search terms. The concerns the participant raised were clearly observed as they embarked in the second part of the mini-usability test. While they searching for both Vietnam War and War World II. The user's input were the following: World War 2, World War II, and Vietnam War. The output included titles with individual words from the input. Many of the results did not have high relevancy to the original search input. As I discovered in the Heuristic analysis many of the results were repeats of the same articles which cluttered the output results. Although, the list of Archived materials exceed the sizing of the screen and there was no function to expand or collapse the archived years, the archive list was used efficiently by the participant. My suggestions for the webpage is to remove the "moving" images and articles to simplify the layout and to not produce distractions for the users. The next suggestion is to put in advance search query capabilities. User's without Boolean search knowledge will have to shift through an immense amount of material before they will find relevant results. The last suggestion, is to produce "similar articles" through the usage of metadata terms linked to each article.
